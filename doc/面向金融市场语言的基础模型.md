# 摘要
以大语言模型（LLM）为代表的大规模预训练范式的成功，为时间序列基础模型（TSFM）的发展提供了启发。但这类模型在金融蜡烛图（K线）数据上的应用仍十分受限，表现往往不及非预训练架构。此外，现有TSFM通常忽略了波动率预测、合成数据生成等关键下游任务。为解决上述局限，我们提出Kronos：一款专为金融K线建模打造的统一、可扩展预训练框架。Kronos引入了专用分词器，可将连续市场信息离散化为token序列，同时保留价格动态与交易活动模式。我们基于来自全球45家交易所、超过120亿条K线记录的大规模跨市场语料，采用自回归目标对Kronos进行预训练，使其能够学习精细的时序与跨资产表征。Kronos在各类金融任务的零样本设置下表现优异：在基准数据集上，Kronos将价格序列预测的RankIC较头部TSFM提升了93%，较最优非预训练基线提升了87%；其波动率预测的MAE降低了9%，合成K线序列的生成保真度提升了22%。上述结果证明，Kronos是一款适用于端到端金融时间序列分析的稳健、通用基础模型。我们的预训练模型已公开发布于：https://github.com/shiyu-coder/Kronos。

# 1 引言
基础模型（FM）的兴起为人工智能领域带来了范式转变，重构了表征学习与下游任务适配的方法论。这一转变的典型代表是大语言模型（LLM）在自然语言处理领域的成功（Brown et al. 2020; Achiam et al. 2023），计算机视觉领域也取得了同步突破（Radford et al. 2021; Kirillov et al. 2023）。

受上述进展启发，基础模型范式近期被拓展到时序数据领域，催生了时间序列基础模型（TSFM）（Garza, Challu, and Mergenthaler-Canseco 2023; Woo et al. 2024; Xiaoming et al. 2025）。其核心目标是构建预训练的、任务无关的架构，作为各类时序分析任务（从预测、异常检测到因果推断）的通用骨干网络，从而大幅减少各应用领域定制化模型设计的需求。

在这一不断拓展的研究领域中，金融市场因其固有的数据丰富性、高频观测特性以及复杂的非平稳时序动态，成为TSFM极具价值也极具挑战性的应用场景。该领域的核心是K线序列，即从蜡烛图衍生的多变量时间序列，记录固定时间间隔内的开盘价（Open）、最高价（High）、最低价（Low）、收盘价（Close）以及交易量（Volume）和交易额（Amount/ Turnover），即OHLCVA数据。这些序列构成了一种高度紧凑、信息密度极高的“语言”，市场参与者通过它解读价格走势、波动率区间、流动性变化以及集体情绪（Nison 2001），因此K线数据是众多算法交易策略、投资组合优化方案与风险管理系统的基础。

但将通用TSFM应用于金融K线数据面临显著挑战，主要源于两点核心因素：第一，K线序列具备独特的统计特性，例如低信噪比、强非平稳性，以及OHLCVA属性之间复杂的高阶依赖关系（Zhang and Hua 2025; Baidya and Lee 2024），往往与通用TSFM的归纳偏置不匹配；第二，主流TSFM研究对金融领域的覆盖不足：现有大多数TSFM的预训练语料中，金融序列仅占很小比例（Das et al. 2024; Gao et al. 2024; Xiaoming et al. 2025），而量化金融领域的关键下游任务（包括波动率估计、合成序列生成、风险管理等）大多未被覆盖。这些因素导致了本研究通过实验验证的重要结论：通用TSFM在金融任务上的表现往往不及专用非预训练模型（例如iTransformer (Liu et al. 2023)），也无法在更广泛的量化金融场景中实现泛化。

为解决上述不足，我们推出了专为金融K线数据设计的统一、可扩展预训练框架Kronos。Kronos采用专用分词器，将连续的多变量K线输入离散化为紧凑的token序列，保留关键的价量交互特征。随后我们基于来自全球45个市场、覆盖7种时间粒度、超过120亿条K线记录的大规模异构语料，对Kronos进行自回归预训练。

我们通过一系列量化金融任务的综合实验验证了Kronos的效果，图1展示了整体性能汇总。在价格序列预测这一核心任务上，Kronos达到了新的最优水平，将RankIC较头部TSFM提升了93%，较最优非预训练基线提升了87%。此外它还展现出极强的通用性：波动率预测的MAE降低了9%，合成K线生成的保真度提升了22%。这些发现凸显了我们方法的广泛有效性，也证明Kronos具备作为解读金融市场复杂“语言”的稳健基础模型的潜力。

我们的主要贡献可总结如下：
• 我们提出了一种面向金融K线数据的新型建模框架，可学习层级表征。该框架配备专用分词器，将每条多变量K线记录量化为结构化的双组分（粗粒度与细粒度）token，同时搭配定制化的自回归目标，可按顺序预测这些子token。这种从粗到细的预测机制让Kronos能够显式建模多尺度市场动态。
• 我们对不同容量的Kronos模型系列进行了大规模预训练。预训练基于来自全球45家交易所、超过120亿条K线记录的多元大规模金融语料完成，这是学习稳健、可泛化的市场表征的基础，也是模型性能优异的核心支撑。

# 2 预备知识
设$D$维向量$\mathbf { x } _ { t } \in \mathbb { R } ^ { D }$表示离散时间$t$的K线观测值，包含$D$个核心金融指标。本研究中我们固定维度$D = 6$，对应OHLCVA属性（开盘价、最高价、最低价、收盘价、交易量、交易额）。该输入选择的逻辑详见附录H（Q1）。给定历史序列$\mathbf { x } _ { 1 : T } ~ = ~ \left( \mathbf { x } _ { 1 } , \mathbf { x } _ { 2 } , \ldots , \mathbf { x } _ { T } \right)$，我们的目标是预测后续$H$个观测值$\hat { \mathbf { x } } _ { T + 1 : T + H } =$ $\left( \hat { \mathbf { x } } _ { T + 1 } , \hat { \mathbf { x } } _ { T + 2 } , \hdots , \hat { \mathbf { x } } _ { T + H } \right)$。

Kronos不会直接处理原始连续输入，而是先通过可学习codebook将每个多元观测值$\mathbf { x } _ { t }$量化为离散令牌$b _ { t }$。因此原始序列$\mathbf { x } _ { 1 : T } ~ = ~ ( \mathbf { x } _ { 1 } , \ldots , \mathbf { x } _ { T } )$会被映射为$\mathbf { b } _ { 1 : T } = ( b _ { 1 } , \ldots , b _ { T } )$。预测任务随即简化为自回归令牌序列建模问题：

$$
p \left(\mathbf {b} _ {T + 1: T + H} \mid \mathbf {b} _ {1: T}\right) = \prod_ {h = 1} ^ {H} p \left(b _ {T + h} \mid \mathbf {b} _ {1: T + h - 1}\right). \tag {1}
$$

这种离散化建模方式天生具备可扩展性，且可以自然扩展到其他可适配生成式框架的任务，比如合成数据生成、波动率预测。

# 3 研究方法
Kronos将金融K线序列抽象为一种离散语言，通过如图2所示的两阶段框架实现该逻辑：（1）K线令牌化；（2）自回归预训练。第一阶段，我们设计了专用的基于Transformer的令牌化器，通过可学习codebook将连续多元K线序列量化为对应的离散令牌序列。每条K线条目（OHLCVA）被视为独立实例，量化为一个离散令牌。每个令牌由粗粒度子令牌和细粒度子令牌构成。我们通过分层重构损失实现该特性，该损失会明确迫使子令牌建模不同层级的信息，从而构建从粗到细的信息层级。第二阶段，我们在这些令牌化序列上预训练一个仅解码器的自回归Transformer，采用标准的下一个令牌预测目标，基于给定的历史上下文按顺序预测未来每个时间步的两个层级子令牌。这种统一的“量化-生成”范式让Kronos可以构建高保真、分层的市场动态表征，为下游量化分析提供了稳固的基础。

为强化每个令牌内部的从粗到细结构，我们采用组合目标训练令牌化器，该目标结合了分层重构损失和BSQ的承诺损失：

$$
\mathcal {L} _ {\text {tokenizer}} = \mathcal {L} _ {\text {coarse}} + \mathcal {L} _ {\text {fine}} + \lambda \mathcal {L} _ {\text {quant}}, \tag {2}
$$

其中$\lambda$为平衡超参数。各损失项定义如下：
• $\mathcal { L } _ { \mathrm { coarse } } = \mathbb { E } \big [ \| { \mathbf { x } } - E _ { \mathrm { dec } } ( { \mathbf { b } } ^ { c } ) \| ^ { 2 } \big ]$：该损失训练粗粒度子令牌${ \bf b } ^ { c }$生成低保真重构结果。
• $\mathcal { L } _ { \mathrm { fine } } = \mathbb { E } \big [ \| \mathbf { x } - E _ { \mathrm { dec } } ( \mathbf { b } ) \| ^ { 2 } \big ]$：该损失使用完整令牌$\mathbf{b}$评估高保真重构效果。
• ${ \mathcal { L } } _ { \mathrm { quant } }$为BSQ（Zhao, Xiong, and Krahenbühl 2024）提出的量化损失，用于正则化学习过程。它会惩罚连续隐向量$\boldsymbol { \xi }$与其二进制编码$\mathbf{b}$之间的L2距离，对齐编码器输出与学习到的codebook，从而保证训练稳定。

这种分层重建目标是我们设计的核心。通过优化$\mathcal { L } _ { \mathrm { c o a r s e } }$，粗粒度子标记${ \bf b } ^ { c }$会学习捕捉输入的核心结构。相应地，在优化$\mathcal { L } _ { \mathrm { f i n e } }$的过程中，细粒度子标记$\mathbf { b } ^ { f }$会被引导编码用于优化粗粒度近似结果所需的残差信息。已有研究表明，从粗到细的解码顺序可以提升生成质量（Wang等人，2025）。我们的方法无需识别并优先解码本身包含粗粒度信息的token，而是在量化阶段就将这种分层结构明确植入token中，确保第一个子标记始终代表粗粒度信息，为后续的autoregressive建模阶段建立所需的条件依赖关系。

# 分层自回归建模

在tokenization阶段结束后，我们使用仅解码器结构的Transformer对生成的离散序列建模，该模型记为$E _ { \mathrm { a r } }$，采用causal-attention机制确保每个时间步的预测仅依赖历史context。核心目标是估计token序列$\mathbf { b } = \{ b _ { 1 } , \ldots , b _ { T } \}$的联合分布，公式1的简化形式可推导为：

$$
p (\mathbf {b}) = \prod_ {t = 1} ^ {T} p \left(b _ {t} \mid \mathbf {b} _ {<   t}\right), \tag {3}
$$

其中$\mathbf { b } _ { < t }$表示时间步$t-1$及之前的所有前置token。

基于我们的分层token设计，每个token的结构为$b _ { t } = [ b _ { t } ^ { c } , b _ { t } ^ { f } ]$，我们使用链式法则进一步分解条件概率，以显式捕捉天然存在的从粗到细依赖关系：

$$
p \left(b _ {t} \mid \mathbf {b} _ {<   t}\right) = p \left(b _ {t} ^ {c} \mid \mathbf {b} _ {<   t}\right) \cdot p \left(b _ {t} ^ {f} \mid \mathbf {b} _ {<   t}, b _ {t} ^ {c}\right). \tag {4}
$$

这种形式让模型可以先预测粗粒度子标记，作为后续生成细粒度残差子标记的框架。因此预训练目标简化为在该分层分解方式下最大化观测序列的对数似然。

如图2（右）所示，自回归流程首先为每个时间步构建统一的输入向量。具体来说，在时间步$i$，子标记$b _ { i } ^ { c }$和$b _ { i } ^ { f }$通过两个独立的embedding层分别映射为向量表示$e _ { c } ( b _ { i } ^ { c } )$和$e _ { f } ( b _ { i } ^ { f } )$。随后这些embedding会被拼接并做线性映射，生成融合后的输入向量：

$$
\mathbf {v} _ {i} = W _ {\text {f u s e}} \left(\left[ e _ {c} \left(b _ {i} ^ {c}\right); e _ {f} \left(b _ {i} ^ {f}\right) \right]\right), \tag {5}
$$

$$

### 细粒度子标记预测
为了建模公式(4)中的条件依赖关系，需要用预测得到的粗粒度子标记$\hat { b } _ { t } ^ { c }$更新context。训练过程中，我们使用模型上一步的预测结果$\hat { b } _ { t } ^ { c }$（从预测分布$p ( b _ { t } ^ { c } | \mathbf { b } < t )$中采样得到），而非使用真实值子标记（即teacher-forcing机制）。我们发现这种采样策略可以缓解exposure bias，提升模型鲁棒性，让训练分布和多步推理的自回归特性更好地对齐——多步推理阶段无法获取真实token。我们采用cross-attention机制，其中$\hat { b } _ { t } ^ { c }$的embedding作为query，历史隐状态$\mathbf { h } _ { t }$提供key和value，结果通过第二个线性头$W _ { f }$映射：

$$
\begin{array}{l} \mathbf {h} _ {t} ^ {\text {u p d a t e}} = \operatorname {C r o s s A t t n} (q = e _ {c} (\hat {b} _ {t} ^ {c}), k = v = \mathbf {h} _ {t}) \tag {7} \\ p \left(b _ {t} ^ {f} \mid \mathbf {b} _ {<   t}, b _ {t} ^ {c}\right) = \operatorname {s o f t m a x} \left(W _ {f} \mathbf {h} _ {t} ^ {\text {u p d a t e}}\right) \\ \end{array}
$$

整体训练目标$\mathcal { L } _ { \mathrm { a r } }$是数据的负对数似然，对两个预测步求和得到：

$$
\mathcal {L} _ {\mathrm {a r}} = - \mathbb {E} _ {\mathbf {b} \sim \mathcal {D}} \sum_ {t = 1} ^ {T} \left[ \log p \left(b _ {t} ^ {c} \mid \mathbf {b} _ {<   t}\right) + \log p \left(b _ {t} ^ {f} \mid \mathbf {b} _ {<   t}, b _ {t} ^ {c}\right) \right] \tag{8}
$$

其中$\mathcal { D }$代表数据分布。

### 模型训练
受LLM中观测到的缩放定律（Kaplan et al. 2020）启发，我们训练了三种参数规模递增的Kronos变体，最大参数量接近5亿，可在性能和推理成本之间实现平衡。详细的模型配置见表1。考虑到资源限制和实际部署场景，我们将最大上下文长度限制为512 tokens。尽管如此，通过使用不同频率的K-line数据，该设计完全支持任意预测周期：例如使用1分钟数据做短期预测，使用日线数据做周度或月度预测。完整的训练细节见附录C。

### 推理
推理阶段，我们以自回归方式生成未来token序列，这一过程与文本生成类似。该过程的随机性通过temperature scaling、top-p（核）采样等标准技术控制（Holtzman et al. 2019）。从logits $\mathbf{z}$ 中采样token $i$ 的概率计算公式为 $p_i \propto \exp(z_i/T)$，其中$T$为温度系数。对于高精度要求的任务，可通过生成多条未来轨迹（即Monte Carlo rollouts），对解码后的连续值取平均来生成更稳定的预测结果，以此提升预测准确率。我们的实验证明，该方法可稳定提升预测质量。

# 主要结果
### 预测任务
图4(a-c)展示了三项预测任务的结果，Kronos在所有任务中均取得了一致的SOTA表现。具体而言，在价格序列预测任务中，Kronos的RankIC相比性能最强的TSFM基线大幅提升93%，相比最优非预训练模型提升87%。此外，随着模型规模扩大，这些任务的表现持续提升，从实验角度验证了时序基础模型的缩放定律（Yao et al. 2024）。

### 生成任务
依照业界通用做法（Yoon, Jarrett, and Van der Schaar 2019），我们从多样性、保真度和实用性三个维度评估合成数据的质量。为评估多样性（即生成样本对真实数据分布的覆盖程度），我们采用两种可视化方法：使用t-SNE将原始和合成数据投影到二维空间，以及通过核密度估计（KDE）对比二者的分布。如图5和附录F所示，t-SNE图显示Kronos生成的合成数据与原始数据空间的重叠度更高，KDE图也验证了二者的分布相似度更高。

定量评估方面，我们使用判别分数评估保真度（即数据真实性），该指标衡量分类器区分原始样本和合成样本的难度。我们还通过“合成数据训练、真实数据测试（TSTR）”范式评估实用性（即合成数据用于训练下游模型的有效性）：在合成数据上训练预测模型，再在由真实数据组成的测试集上评估其IC和RankIC表现。如图4(d)所示，Kronos在保真度和实用性上均取得最优表现，且这种优势会随着模型规模扩大进一步提升。

### 投资模拟
为验证Kronos在真实投资场景下的表现，我们在中国A股市场模拟了仅做多的投资策略：根据各模型的预测信号排名选取前k只股票构建投资组合。如图4(e)所示，Kronos表现优于所有其他基线，取得了最高的年化超额收益（AER）和信息比率（IR），证明该模型可将其领先的预测准确率有效转化为实际投资收益。

# 消融实验
我们开展了消融实验以验证核心设计选择的有效性，重点围绕两个问题展开：（Q1）与其他替代方案相比，我们的建模范式的有效性；（Q2）词汇表大小的影响。关于tokenizer的额外消融实验结果见附录E。

## 建模范式分析
为解答Q1，我们在保持参数量相当的前提下，将Kronos与预测空间、目标函数不同的变体进行了对比（见表2）。这些架构变体的详细说明见附录D。我们测试了两个连续空间模型：Direct-AR（基于MSE的回归基线模型）和Prob-AR。参照已有研究（Yao等人，2024），Prob-AR使用Student-t混合分布来更好地建模重尾数据分布。结果表明，我们的离散空间模型性能明显优于这些连续空间备选方案。我们还发现，并行预测subtoken的变体Kronos-Parallel性能比我们的序列化方法差，这证明了对subtoken依赖关系建模的重要性。这些结果验证了我们的离散序列化建模框架是该领域更有效的解决方案。

## 词汇表大小的影响
为解答Q2，我们研究了词汇表大小对模型性能的影响。如图6所示，增大词汇表大小可同时提升重构质量和预测准确率。更大的词汇表能提供更细粒度的表示，降低量化误差。关键在于，这种表示精度的提升可以转化为更好的预测效果。这一发现与视频生成领域的观察结果一致：对于LFQ、BSQ等量化技术，更大的词汇表可以提升生成质量（Zhao, Xiong, and Krahenbühl 2024; Yu et al. 2023）。

# 测试时缩放
我们的概率生成框架的一个显著优势是无需重新训练模型，即可在推理阶段提升预测准确率。通过利用随机采样，Kronos可以基于同一个context window生成多条不同的未来轨迹。我们通过对越来越多采样路径的结果取平均，研究了这些预测集成的效果。图7展示了预测任务性能随样本数变化的情况。结果表明，随着集成中包含的样本数增加，IC和RankIC均持续提升。对多条路径取平均可以缓解生成过程固有的随机性，降低预测方差，得到更鲁棒、更稳定的估计结果。这一特性提供了一种权衡方案，让从业者可以在推理阶段的计算成本和预期预测准确率之间做平衡。

# 5 结论
在本研究中，我们提出了Kronos，一款专门针对金融K线序列设计的基础模型。Kronos采用新颖的两阶段框架：首先基于实例的tokenizer将连续的市场数据离散化为从粗到细的分层token，之后通过大型自回归Transformer对这些token建模。全面的实验评估表明，Kronos在价格序列预测，以及合成K线生成、波动率预测等其他相关应用中均建立了新的state-of-the-art基准，性能显著优于现有TSFM和其他基线模型。这些结果表明，Kronos可为各类量化金融应用提供鲁棒、通用的基础支持。

# A 相关工作

# 时间序列Tokenization

最近，基于token的大型模型的成功激发了人们对连续时间序列离散化的日益浓厚兴趣。这一tokenization流程对于调整此类架构以适配时间序列分析任务至关重要，但目前该领域的专门研究仍较为匮乏。Chronos（Ansari等人，2024）等早期研究采用缩放和均匀量化方案，而TOTEM（Talukder、Yue与Gkioxari，2024）则使用向量量化变分自编码器（Vector Quantized Variational Autoencoder, VQ-VAE）（Van Den Oord、Vinyals等人，2017）——这是一种将编码器输出映射到已学习离散隐编码的开创性方法——实现基于码本的tokenization。鉴于该领域尚处于发展初期，我们从发展更成熟的视觉tokenization领域汲取灵感。除基础VQ-VAE外，相关创新还包括无查找量化（Lookup-Free Quantization, LFQ）（Yu等人，2023），该方法通过隐式码本无需显式查找即可实现高保真重建。二元球面量化（Binary Spherical Quantization, BSQ）（Zhao、Xiong与Krahenbühl，2024）利用球面投影优化隐式码本，可实现指数级增长的词表，相比LFQ具备有界量化误差和更优的可训练性。此外，索引反向传播量化（Index Backpropagation Quantization, IBQ）（Shi等人，2025）通过让所有码本条目可微解决码本坍塌问题，支持大规模码本与视觉编码器的稳定联合优化。尽管这些方法主要为视觉数据设计，也可应用于通用多变量时间序列的离散化。

# 通用时间序列基础模型

时间序列分析的范式近来已被时间序列基础模型（Time Series Foundation Models, TSFMs）重塑，这类模型受大语言模型利用大规模预训练Transformer取得成功的启发而诞生。这些模型在庞大的多域语料库上训练（部分语料包含超千亿数据点），在通用预测基准上实现了出色的zero-shot与few-shot性能。这种通用性由多样化的架构支撑，包括：仅解码器模型，如Lag-Llama（Rasul等人，2023）、TimesFM（Das等人，2024）、Timer（Liu等人，2024）、Time-MoE（Xiaoming等人，2025）以及Sumdial（Liu等人，2025）；仅编码器框架，如MO-MENT（Goswami等人，2024）和Moirai（Woo等人，2024）；编码器-解码器结构，如TimeGPT（Garza、Challu与Mergenthaler-Canseco，2023）；以及为多任务学习修改Transformer块的模型，如UniTS（Gao等人，2024）。在输入层面，它们采用通用表示方案，例如直接值分块（如TimesFM（Das等人，2024）、MOMENT（Goswami等人，2024））、将值量化到固定词表（如Chronos（Ansari等人，2024）），或将连续时间点视为token（如Timer（Liu等人，2024））。部分模型还扩展支持概率预测（如Lag-Llama（Rasul等人，2023）、Moirai（Woo等人，2024）、Chronos（Ansari等人，2024）和Sumdial（Liu等人，2025））。

然而，支撑其在广泛基准上取得成功的通用性，反而成为了这类模型在专业领域的局限性。为提供具体对比，我们在表3中总结了主流TSFM的关键属性。从表中可观察到一个重要结论：这些通用模型的预训练语料中金融数据占比极低，大多数模型分配给该领域的数据占比不到$1\%$。这种数据失衡意味着金融K线序列独特的结构属性、非平稳性和复杂动态在预训练过程中基本被忽略或被平均，通常导致模型在金融任务上表现不佳。为填补预训练环节的这一根本空白，我们推出Kronos，这是一款完全基于专属大规模金融K线数据语料库从头构建的基础模型。

# 金融时间序列基础模型

专门面向金融时间序列的基础模型开发是一个新兴但快速发展的领域。相关工作可分为两大主流方向：第一个方向聚焦通用金融时间序列，包括K线数据。例如PLUTUS（Xu等人，2024）提出了可逆嵌入和多尺度时间注意力机制，在大规模数据集上预训练以挖掘市场规律；DELPHYNE（Ding、Mittal与Gopal，2025）专为抵消非金融数据带来的负迁移而设计。尽管这些研究前景可观，但均未开源代码或模型，无法进行直接的实证对比。第二个方向针对订单流数据，MarketGPT（Wheeler与Varner，2024）和MarS（Li等人，2024）等模型可作为生成引擎生成逼真的市场模拟数据。这些开创性工作验证了领域专属预训练的价值，但K线数据比订单流适用范围更广，它在所有市场都可轻松获取，且适配订单流数据通常无法覆盖的多种时间周期。尽管K线数据的核心地位突出，目前仍明显缺乏一款通用、开源的K线分析基础模型。我们推出Kronos来填补这一空白，它是专为金融K线数据设计的统一、可扩展框架。

### 缺失值处理
我们采用分字段策略处理缺失值，这类缺失值通常表示为`NaN`（非数字）或`Inf`（无穷大）。
- **价格字段（开盘价、最高价、最低价、收盘价）**：对于价格相关字段，我们将缺失值视为硬边界。受TimeMOE（Xiaoming等，2025）启发，我们在每个缺失价格值的位置将时间序列切分为连续的有效子序列。该方法无需无依据的插补，可确保每个生成的片段都保持内部时序完整性。
- **成交量与成交额字段**：与之不同的是，成交量和成交额字段主要作为辅助协变量，我们用0填充这类字段的缺失值。为了提升模型对稀疏或不可用的成交量类数据的鲁棒性，我们引入了一种正则化技术：训练期间，我们会随机将5%输入样本的成交量和成交额设为0，这能促使模型学习仅通过价格信息即可做出有效预测。

### 低质量片段过滤
除了处理离散缺失值之外，我们的流水线还会系统性地识别并移除整段的低质量数据。这一目标通过多阶段过滤流程实现，其中容忍阈值会根据数据的时间频率动态调整，详见表4。该流程在算法1中被规范化，包含以下步骤：
- **结构断裂分段**：初始过滤阶段基于显著的价格不连续性对序列进行切分。我们通过计算前一根K线的收盘价与当前K线的开盘价之间的相对价格跳空 $( | \mathrm { o p e n } _ { t } / \mathrm { c l o s e } _ { t - 1 } - 1 | )$ 来识别这类断裂。如果跳空幅度超过对应时间频率的阈值，就对序列进行切分。该步骤可有效隔离合约移仓、股票拆分或股息发放等事件带来的伪影。
- **流动性不足时段过滤**：在上一步得到的每个片段中，我们筛选出持续流动性不足的时段。如果某根K线的成交量为0或接近0，则判定为流动性不足K线。如果连续流动性不足的K线数量超过对应时间频率的阈值，对应时段将被标记为无效。
- **价格停滞过滤**：我们采用类似方法过滤价格停滞时段，即收盘价在较长一段时间内保持不变的情况，这通常意味着可能存在数据源问题或市场不活跃。如果价格停滞的持续时长超过对应时间频率的容忍阈值，也会被标记为无效时段。
- **最终片段验证**：在标记完所有流动性不足和价格停滞时段后，我们会在这些标记区域的边界处进一步切分初始片段。最终，只有满足表4中对应时间频率最小长度要求 $\Theta _ { \mathrm { m i n \mathrm { \tiny ~ l e n } } }$ 的子片段才会被保留，进入最终的预训练数据集。这可确保每个样本的长度足够支撑有效的模型学习。

# C 实现细节
在本节中，我们将进一步说明Kronos的实现细节，涵盖数据预处理、模型架构，以及训练和推理的配置。

# 输入预处理
每个输入K线序列 $\mathbf { x } = \left( \mathbf { x } _ { 1 } , \mathbf { x } _ { 2 } , \ldots , \mathbf { x } _ { T } \right)$（其中 $\mathbf { x } _ { t } \in \mathbb { R } ^ { \tilde { D } }$）在传入tokenizer之前，会经过两步归一化处理。首先，我们对每个$D$维特征（例如开盘价、最高价、最低价、收盘价、成交量、成交额）独立应用z-score归一化。其次，为了降低极端异常值对训练稳定性的潜在影响，归一化后的值会被裁剪到 $[ - 5 , 5 ]$ 区间。该流程确保所有输入特征的尺度一致，同时保留模型对异常数据点的鲁棒性。

# 模型架构
**时序嵌入**：为了捕捉金融市场固有的周期性模式，例如日内、周度和月度季节性（Ozenbas等，2008；Kohli和Kohers，1992），我们引入了可学习的时序嵌入。我们为每条K线提取5个时间相关特征：日内分钟、日内小时、周内第几天、月内第几天、年内第几月。每个特征通过专用的嵌入层映射为稠密向量。这些时序嵌入求和后，会加到对应token的输入表征上，为模型提供显式的时序上下文。

**K线Token化**：tokenizer的自编码器被设计为轻量型。编码器和解码器各包含3层Transformer层，模型维度为256，前馈网络维度为512，注意力头数为4。参照BSQ的官方开源实现，我们将核心量化超参数配置如下：承诺权重 $\beta = 0 . 0 5$，熵惩罚权重 $\gamma _ { 0 } = 1 . 0$、$\gamma = 1 . 1$，全局熵缩放系数 $\zeta = 0 . 0 5$。目标函数中量化损失的平衡超参数 $\lambda$ 设为1。为了让熵计算更易处理，量化分组大小设为5。

**Transformer块架构**：为了编码数据的序列特性，我们采用带旋转位置嵌入（RoPE）（Su等，2024）的因果自注意力机制，该机制可注入相对位置信息。注意力运算的公式如下：

$$
\operatorname {A t t e n t i o n} (Q, K, V) = \operatorname {C a u s a l M a s k} \left(\frac {Q ^ {\prime} \left(K ^ {\prime}\right) ^ {T}}{\sqrt {d _ {k}}}\right) V \tag {9}
$$

其中$d_k$为键向量的维度，CausalMask用于避免模型关注未来位置。矩阵$Q'$和$K'$是经过RoPE变换后的原始查询矩阵和键矩阵。此外，我们采用前置层归一化（Pre-LN）(Xiong et al. 2020)来提升训练稳定性，出于计算效率和性能考量，具体使用均方根层归一化（RM-SNorm）(Zhang and Sennrich 2019)实现。

# 训练配置

我们为每种尺寸的模型精心选择训练超参数，以确保预训练过程稳定。随着模型规模增大，我们降低峰值学习率和dropout概率，同时增大权重衰减系数。我们使用AdamW优化器(Loshchilov and Hutter 2017)，以及带线性预热阶段的余弦学习率调度策略。在前15000个训练步中，学习率从峰值的10%逐步上升。各模型变体的具体超参数设置详见表5。

# 推理超参数

# 附录D 实验设计与实现

本节我们将介绍用于Kronos评估的完整实验设计与实现细节。我们首先概述核心评估任务及其对应的指标，随后介绍用于对比的所有基线模型及其具体配置，最后详细说明每个实验任务的实现方式，涵盖评估所用的数据集、参数和具体实验协议。

# 任务与评估指标

我们在量化金融领域的多个核心任务上评估Kronos的性能，各任务及其对应的评估指标如下：

• 价格序列预测：我们评估模型预测未来价格序列的能力，使用预测值与真实值之间的信息系数（IC）和秩信息系数（RankIC）衡量性能。
• 收益率预测：同理，我们评估模型预测资产收益率的能力，同样使用IC和RankIC作为指标衡量预测精度。
• 已实现波动率预测：我们使用模型的高频预测结果估计已实现波动率，使用平均绝对误差（MAE）和决定系数（$R^2$）评估估计的准确性。
• 合成K线生成：遵循时间序列生成领域的常规做法(Yoon, Jarrett, and Van der Schaar 2019)，我们从三个维度评估合成K线序列的质量：多样性，评估生成样本对真实数据分布的覆盖程度；保真度，评估合成样本是否与真实样本难以区分；实用性，评估合成数据在下游预测任务上是否能达到与真实数据相当的效果（即“合成数据训练、真实数据测试”范式）。
• 投资模拟：为衡量模型预测结果的实际适用性，我们执行回测模拟，使用年化超额收益率（AER）和信息比率（IR）报告性能。

# 基线模型与配置

为实现严谨的评估，我们选取了共25个基线模型与Kronos进行对比。这些模型均选自已有研究（如(Xiaoming et al. 2025; Wang et al. 2024b; Yuan and Qiao 2024)），覆盖了不同范式下的各类成熟方法和当前最优方法，分为四个不同类别：

• 全样本微调时间序列模型：该类别包含无需预训练、在特定下游任务上从零开始训练的现代时间序列模型，包括TimeXer(Wang et al. 2024c)、Times-Net(Wu et al. 2022)、TimeMixer(Wang et al. 2024a)、PatchTST(Nie et al. 2022)、非平稳Transformer（NSTransformer）(Liu et al. 2022)、DLinear(Zeng et al. 2023)、FEDformer(Zhou et al. 2022)和iTransformer(Liu et al. 2023)。
• 零样本时间序列模型：该类别包含面向通用时间序列分析的大规模预训练基础模型，基线包括TimeMOE(Xiaoming et al. 2025)、Moirai(Woo et al. 2024)、TimesFM(Das et al. 2024)、Moment(Goswami et al. 2024)和Chronos(Ansari et al. 2024)，我们在零样本设置下评估这些模型的性能。
• 计量经济学波动率模型：针对波动率预测任务，我们加入了成熟的计量经济学模型作为专用基线，即ARCH(Engle 1982)和GARCH(Bollerslev 1986)。
• 生成式时间序列模型：针对K线生成任务，我们将Kronos与代表三种主流生成架构的模型进行对比：DiffusionTS（基于扩散模型）(Yuan and Qiao 2024)、TimeVAE（基于VAE）(Desai et al. 2021)和TimeGAN（基于GAN）(Yoon, Jarrett, and Van der Schaar 2019)。

### 全样本微调时间序列模型
对于所有非预训练深度学习模型，我们使用结合了均方误差（MSE）和信息系数（IC）项的复合损失函数。我们通过实验发现，与仅使用MSE相比，该目标函数能够提升金融任务的预测性能，因为它会直接奖励模型准确捕捉价格变动的方向。损失函数定义如下：

$$
\mathcal {L} = \frac {1}{M \cdot H} \sum_ {i = 1} ^ {M} \sum_ {j = 1} ^ {H} \left(y _ {i, j} - \hat {y} _ {i, j}\right) ^ {2} - \lambda \cdot \frac {1}{M} \sum_ {i = 1} ^ {M} \mathrm {I C} \left(y _ {i}, \hat {y} _ {i}\right) \tag {10}
$$

其中$y _ { i }$和$\hat { y } _ { i }$分别为第$i$个特征的真实序列和预测序列，$M$为特征数量，$H$为预测时域，$\lambda$为平衡超参数，在我们的实验中取值为4。

所有模型训练的batch size为256，采用Adam优化器，学习率为$5 \times 1 0 ^ { - 4 }$。我们最多训练12个epoch，同时引入早停机制：当验证集损失连续3个epoch没有下降时停止训练。为确保对比公平、结果鲁棒，我们为每个模型测试了对应小、大两种模型尺寸的两组超参数，最终选择在验证集上表现最优的配置用于最终评估。对于DLinear模型，我们没有调整模型维度，而是基于其`individual`参数评估两种配置：一种是所有变量共享单个线性层（`individual=False`），另一种是为每个变量训练独立的线性层（`individual=True`）。具体超参数配置详见表7。

对于专用波动率预测基线模型，我们遵循标准计量经济学实践进行模型选择：

### 数据集与生成参数

# 评估指标
• 判别分数：为评估生成数据的保真度，我们采用一个基于LSTM的事后分类器来区分真实序列和合成序列。该分类器包含一个隐藏维度为32的单层LSTM。训练时，我们构建了包含6000个样本的平衡数据集（3000条真实、3000条合成），以及一个相同规模和构成的留出测试集。模型训练20个epoch，batch size为64，采用Adam优化器（学习率$= 0 . 0 0 0 5$）和二元交叉熵（BCE）损失函数。判别分数定义为测试集上的分类错误率，分数越接近0.5说明保真度越高，意味着分类器难以区分生成数据和真实数据。

• 实用性（TSTR）：为衡量合成数据的实际实用性，我们采用“合成数据训练、真实数据测试（Train-on-Synthetic, Test-on-Real，TSTR）”方法。我们训练了一个事后LSTM预测模型，基于历史时间窗口预测未来K步的时间窗口。该模型包含2层隐藏维度为64的LSTM层，仅使用6000条生成的合成序列训练20个epoch，优化器采用Adam（学习率$=$ 0.001），批量大小为64，目标函数为均方误差（Mean Squared Error，MSE）损失。针对15分钟频率数据，回溯窗口和预测窗口分别设为(80, 16)；针对日频数据则设为(30, 5)。训练完成的模型随后在原始真实测试数据上评估，最终实用性得分以预测价格序列的平均信息系数（Information Coefficient，IC）和秩信息系数（Rank Information Coefficient，RankIC）给出。

**投资模拟设置** 为评估Kronos及其他基线模型在真实市场中的实际盈利能力，我们在中国A股市场开展了投资模拟。为简化流程，针对Zero-shot Time Series Models，我们仅选取每个类别中参数量最大的模型进行对比。

**数据** 我们的实证分析采用中国A股市场的日频市场数据，数据来源于量化金融开源框架Qlib平台（Yang et al. 2020）。为提升透明度和可复现性，我们未对数据做任何额外过滤或预处理，直接使用原始未加工的数据。此外，所有回测模拟均在Qlib框架内完成，依托其集成的回测引擎，确保所有受评估模型采用统一、一致的评估协议。

**策略** 我们采用top-k/drop- $\mathbf { \nabla } \cdot n$ 投资组合构建策略。每个交易日，投资池内的所有股票会根据其预测收益信号排序，通过做多排名前$k$的股票构建等权投资组合。为控制换手率和交易成本，每日最多买卖$n$只股票，且所有持仓的最短持有期为5天。

**信号与回测** 预测信号为$H$天周期多步价格预测推导的预期收益。该信号生成流程统一应用于所有受评估模型（包括Kronos和各基线模型），以保证对比公平。对于交易日$t$的任意一只股票，对应模型会先生成后续$H$天的预测收盘价序列，记为$\{ \hat { p } _ { t + i } \} _ { i = 1 } ^ { H }$；随后我们将这些预测价格的算术平均值与当前收盘价$p_t$对比，计算得到我们称为$H$天平均预期收益（$R_{t\rightarrow t+H}$）的信号：

$$
R _ {t \rightarrow t + H} = \frac {\left(\frac {1}{H} \sum_ {i = 1} ^ {H} \hat {p} _ {t + i}\right) - p _ {t}}{p _ {t}} \tag {13}
$$

回测在沪深300（CSI 300）和沪深800（CSI 800）指数的成分股上开展。选择这两个指数的原因是它们代表了中国A股市场的两个核心板块：沪深300由大市值、高流动性的股票组成，而沪深800同时包含大市值和中市值股票，覆盖的市场范围更广，便于我们全面评估模型在不同市场板块中的表现。

**参数与成本** 针对沪深300指数，我们设$k = 5 0$、$n = 5$；针对覆盖范围更广的沪深800指数，我们设$k = 2 0 0$、$n = 1 0$。选择较大的投资组合规模是为了保证分散化，生成更稳定的回测结果，降低个股异质性波动的影响。为保证性能评估符合真实情况，每笔交易适用0.15%的保守交易成本。

# 消融研究基线模型详情

为探究Kronos的架构设计选择，我们为消融研究设计了3种基线变体（表2）。每种变体对应不同的建模范式，便于我们分离出我们提出的离散序列化框架的优势。下文将详细介绍每个模型。

**Direct-AR**：该模型是连续空间下的标准自回归预测基线。给定输入特征序列$\{ x _ { 1 } , \dots , x _ { T } \}$，每个特征向量$\boldsymbol { \dot { x } } _ { t } \in \mathbb { R } ^ { D }$首先通过线性投影映射到更高维的嵌入空间，嵌入序列随后输入Transformer解码器主干处理。该模型的训练目标是基于历史上下文直接预测下一个时间步的值$\hat { x } _ { T + 1 }$，最小化预测值和真实值之间的均方误差（MSE）。该方法是时间序列预测最常用的基于回归的实现形式。

**Prob-AR**：这是运行在连续空间下的概率预测模型。遵循已有研究的常规做法（Yao et al. 2024），Prob-AR不输出点估计，而是预测下一个时间步的概率分布参数。我们使用4个Student-t分布的混合来建模预测分布。服从单Student-t分布的随机变量$x$的概率密度函数（probability density function，PDF）为：

$$
p (x \mid \nu , \mu , \sigma) = \frac {\Gamma (\frac {\nu + 1}{2})}{\Gamma (\frac {\nu}{2}) \sqrt {\pi \nu} \sigma} \left(1 + \frac {1}{\nu} \left(\frac {x - \mu}{\sigma}\right) ^ {2}\right) ^ {- \frac {\nu + 1}{2}} \tag {14}
$$

其中$\nu ~ > ~ 0$ , $\mu \in \mathbb { R }$ , and $\sigma \ > \ 0$分别为自由度、位置参数和尺度参数，$\Gamma ( \cdot )$为伽马函数。该模型使用独立的线性层预测四个分量的对应参数：自由度$( \nu _ { k } )$、位置$( \mu _ { k } )$、尺度$( \sigma _ { k } )$和混合权重$( w _ { k } )$。为保证参数有效性，对$\nu _ { k }$和$\sigma _ { k }$应用softplus变换以确保其为正，对权重$w _ { k }$应用softmax函数以确保其构成合法的概率分布。该模型通过最小化真实值在预测混合分布下的负对数似然（Negative Log-Likelihood, NLL）完成训练。

Kronos-Parallel：该变体是针对Kronos内部序列子令牌生成机制的直接消融实验。它与Kronos使用相同的输入量化规则和离散预测空间，但移除了块内模块。Transformer主干基于输入历史生成上下文向量后，使用单个预测头同时预测下一个时间步两个子令牌的logits。训练目标为两个子令牌的交叉熵损失之和，采用联合优化方式。
