
## 性能优化建议

### 1. 硬件要求

| 资源 | 推荐值 | 备注 |
|------|--------|------|
| GPU显存 | 12GB+ | 分词器和预测器各需6GB |
| CPU | 8核+ | 数据加载并行化需要 |
| 内存 | 32GB+ | 完整数据集加载 |
| 存储 | 50GB+ | 模型和训练日志 |

### 2. 训练时间估计

| 阶段 | CPU模式 | GPU模式 | 说明 |
|------|---------|---------|------|
| 分词器训练 | 12-24小时 | 2-4小时 | 20个epoch |
| 预测器训练 | 8-16小时 | 1-2小时 | 15个epoch |
| **总计** | **20-40小时** | **3-6小时** | 包括验证 |

### 3. 超参数调优

```yaml
# 如果训练不稳定，尝试：
training:
  # 降低学习率
  tokenizer_learning_rate: 0.0001  # 从 0.0002 降低
  predictor_learning_rate: 5e-7    # 从 1e-6 降低
  
  # 增加梯度累积
  accumulation_steps: 2              # 从 1 增加到 2
  
  # 减少批大小
  batch_size: 16                     # 从 32 降低到 16
```

## 常见问题排查

### Q1: 显存不足错误

**症状**: `CUDA out of memory`

**解决方案**:
```yaml
training:
  batch_size: 16  # 降低批大小
  accumulation_steps: 2  # 增加梯度累积
```

### Q2: 验证损失不下降

**症状**: 训练损失下降但验证损失不变

**解决方案**:
```yaml
training:
  tokenizer_learning_rate: 0.0001  # 降低学习率
  adam_weight_decay: 0.05  # 增加正则化
```

### Q3: 预测效果差

**症状**: MAPE > 10%

**解决方案**:
1. 检查数据质量（缺失值、异常值）
2. 增加训练轮数
3. 调整lookback_window大小
4. 检查数据归一化是否正确
